Abstract Factory vs Dependency Injection

Problema: O padrão Abstract Factory cria hierarquias complexas e código verboso, dificultando testes e aumentando acoplamento entre componentes

PROBLEMÁTICO
// Abstract Factory - VERBOSO
public abstract class DatabaseFactory {
    public abstract Connection createConnection();
    public abstract Statement createStatement();
}

public class MySQLFactory extends DatabaseFactory {
    public Connection createConnection() { ... }
    public Statement createStatement() { ... }
}

// Uso complexo
// Precisamos de uma hierarquia de classes
// Difícil de testar componentes isoladamente
// Acoplamento forte entre componentes

DatabaseFactory factory = new MySQLFactory();
Connection conn = factory.createConnection();
Statement stmt = factory.createStatement();
SOLUÇÃO DI
// DI - SIMPLES
public class DatabaseService {
    private final DatabaseConnection connection;
    
    public DatabaseService(DatabaseConnection connection) {
        this.connection = connection;
    }
}

// Configuração externa
DatabaseService service = new DatabaseService(
    new MySQLConnection()
);

// Vantagens:
// 1. Código mais simples e direto
// 2. Fácil de testar com mocks
// 3. Inversão de controle
// 4. Configuração centralizada/externa
// 5. Baixo acoplamento
